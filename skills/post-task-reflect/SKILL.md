# Post-Task Reflection

Run after each completed task. This is lightweight — focus on two things: organizing memory and capturing learnings as skills.

## Step 1: Organize Memory

Read `memory/INDEX.md`, then review and update relevant memory files:

- Did you learn something worth remembering? Write it to an appropriate memory file.
- Are there outdated entries that should be updated or removed?
- Keep INDEX.md accurate after any changes.

## Step 2: Evaluate Skill Capture

Ask yourself about the task you just completed:

- **Was it a multi-step workflow (5+ steps)?** → Create a skill to capture the pattern.
- **Did you solve something novel or tricky?** → Definitely create a skill.
- **Did you use an existing skill that was outdated?** → Update it with `edit`.
- **Is there knowledge worth preserving?** → Save facts to memory, procedures to skills.

**Procedures and workflows belong in skills; facts belong in memory.**

If you decide to create or update a skill, do it now — don't defer it.

## Step 3: Return Summary

Return a brief note of what memory changes were made and whether any skills were created or updated.
